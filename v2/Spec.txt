HaauHei考试系统生成指令集
核心目标：创建一个完全本地运行、无需后端服务器和外部依赖的静态网页考试系统。
一、 项目结构与文件规范

文件构成：系统由 index.html、style.css、script.js 三个核心文件组成。
代码长度限制：为确保可维护性，每个.js或.css文件长度不应超过300行。若逻辑复杂需超出，应将代码模块化并分割为多个文件。
数据存储：所有题目数据以多个JSON文件存储，放置在名为 subjects 的子目录中。
JSON文件命名规则：文件名称必须为4位数字序号，从 0001.json 开始，依次递增（如0002.json, 0003.json）。系统需按此顺序读取，当读取不到下一个序号文件时自动停止。

二、 数据格式规范 (JSON结构)
每个JSON文件代表一个“科目”，需严格遵循以下格式：
{
  "subjectName": "科目名称",
  "questions": [
    {
      "question": "题目内容",
      "options": ["选项A", "选项B", "选项C", "选项D"],
      "correctAnswer": 0,
      "explanation": "答案解释及参考页"
    }
    // ... 更多题目
  ]
}
correctAnswer：为整数，表示正确选项的索引（0对应A，1对应B，以此类推）。

三、 核心功能需求

初始界面与配置：

提供界面让用户选择本次考试的题目数量，预设值为10题。
显示当前题库加载的"科目名称"。

智能出题逻辑：

随机性：出题顺序完全随机，第一题和第二题可能来自不同的JSON文件及文件内的不同位置。每次重新开启系统，出题的选择及次序都应不同。
历史规避机制：系统需在用户浏览器本地 (localStorage) 记录其答题历史。出题时，应优先避免抽取用户近期（例如，10天内）已答对的题目。对于很久之前（超出设定时间范围）答对的题目，可以有一定概率再次出现，以实现复习目的。

考试流程界面：

清晰显示当前题号、总题数、进度条和题目所属科目。
以整洁的样式展示题目和四个选项。
用户选择答案后，才能进入下一题。

考试结果与复盘：

考试结束后，立即统计并显示正确/错误题目数量及正确率。
详细展示每道题的：

用户答案与正确答案。
答案解释。

对正确和错误的题目进行视觉区分（如不同颜色高亮）。
提供“重新开始”按钮，返回初始界面。

四、 技术实现要点

纯前端实现：所有逻辑（读取JSON文件、出题算法、状态管理）均使用JavaScript在浏览器端完成，不依赖任何网络API或服务器。
数据读取：使用 fetch() API 按序号循环读取 subjects 目录下的JSON文件，并合并所有题目。
状态持久化：使用 localStorage 存储用户的答题历史记录。记录应至少包含题目ID、是否正确以及最后一次答对的时间戳。
响应式设计：CSS应确保界面在桌面和移动设备上均有良好的显示效果。
避免使用 eval() 之類會引起browser作安全性限制的function。
不使用外部的js、jquery庫。
所有顯示的中文用繁體字。

五、 开发步骤参考 (基于通用流程归纳)

功能分析与设计：明确上述需求，规划文件结构与模块。
创建基础结构：建立项目文件夹，包含 index.html, style.css, script.js 和 subjects/ 目录。
实现数据层：编写能正确循环读取并解析序号JSON文件的JavaScript函数。
实现核心逻辑：编写随机抽题算法，并集成本地历史记录存储与读取逻辑。
构建用户界面：分别实现欢迎屏幕、考试屏幕和结果回顾屏幕的HTML与CSS。
连接界面与逻辑：为按钮绑定事件，确保用户交互能触发状态改变和界面更新。
测试与优化：使用多个JSON题目文件进行全流程测试，包括出题随机性、历史记录功能等。

通过遵循以上指令集，您可以构建出一个符合要求的、自包含的“HaauHei”本地考试系统。
